Reading Notes

移动应用的设计与开发

记得是在去北京的车上读完这本书的，从出版时间上看或许稍稍过时。但阐述的移动应用设计的基本原则并不过时，全面，抽象，也涵盖一定量的设计模式，方法论以及例子，但这些并不是主体。回想起来，文字间每时每刻都透露出的那种用户第一体验至上的感觉有点打动人。用问题健壮想法，那是创新设计课刚开始的时候，andrew在他邮件里分享给我们的，那时最开始的头脑风暴，大家也不免互相质疑并且对方很可能没办法为新鲜的想法打消一个个质疑，导致低效。半年多了，有点怀念。

Part1 应用环境

1. 背景环境

Defination: 帮助用户更好地理解周围环境，共鸣。
eg：“这条信息为我提供了背景环境，使我更好地理解这一刻对我有什么意义”
“站在柏林墙的遗迹前阅读手机上关于它的历史”

2. 归属环境

a. 所处位置，物理环境
b. 访问时所用的设备，媒体环境
c. 当前的思维状态，情态环境

Part2 用问题健壮想法

在用户可能遇到的各种应用环境中去审视应用程序：

  • 用户是谁？能假设or预测他们有怎样的行为？
  • 发生了什么？用户在什么环境下最能理解你想展现的内容？
  • 用户何时交互？在家赋闲or工作忙里偷闲or等车无聊 etc
  • 用户身在何处？公共场所or个人空间，室内or室外，白天or晚上
  • 用户为何使用你的应用程序？以用户的当前状况，他可以得到什么价值？

  • 用户怎样使用移动设备？拿在手中or放在口袋中，姿势静止or多变

  • Some regulations

先定义用户环境，找出用户目标。Then，尝试理解用户的应用环境是如何改变其目标的。

Part4 视觉设计

  • 主旨
  • 观感
  • 布局：在观感之前，从信息架构中找到问题答案
  • 颜色：P127
    • 调色板：
      • 顺序：三色。第一种选品牌颜色，第二第三为一的补色
      • 最优化：从一张图片中提取常见色
      • 灵感
  • 字体：
    • 高对比度
    • sans-serif无尘先 用于导航or紧凑区域； serif衬线 显示长篇文字
    • 左右留空白 3,4个字符
    • 用文本标题分割内容
    • 缩短段落长度，2-3句子per段

Part5 mobile web app VS native app

When to create native app

  • 收费
  • 游戏
  • 使用特定位置
  • 摄像头
  • 加速度计
  • 访问文件系统
  • 离线用户

Part6 Some valuable questions about USER TEST

应用环境测试 P286

  • 能否迅速加载
  • 设备物理特性能否照计划使用？旋转，按键，切视图 etc
  • 应用能否正确中断when calling
  • 设备中断时会发生什么？用户能否恢复会话
  • 如果处于离线模式，是否会提示错误？
  • 如果大量使用数据，且高速移动（在不同移动发射塔切换），应用能否正常工作？
